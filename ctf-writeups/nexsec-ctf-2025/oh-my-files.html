OhMyFiles #1

This challenge was a straightforward forensic starting point. The task was to obtain the provided disk image and calculate its SHA-256 value correctly. Even though the answer itself was simple, the challenge introduced the case background and the main victim context that would matter for the later OhMyFiles tasks.

Forensics NexSec CTF 2025 Disk Image SHA-256
Main task

Download the forensic disk image and obtain the SHA-256 hash of the evidence file.

Core idea

Start with the supplied case material, retrieve the image safely, then verify it using a hash.

Challenge overview

The first thing I did was review the material provided by the challenge. From there, the workflow became clear: download the files, understand the short case context, install a torrent client to handle the large image, and finally calculate the SHA-256 value of the evidence file.

Step 1: Download the attached files

I started by downloading the files attached in the challenge. This gave me the initial materials needed for the investigation, including the guidance for obtaining the disk image.

Step 2: Review the provided files

After opening the files, I focused on two parts: the installation guide and the short incident report. The installation guide explained why a torrent client was the better option for this task, while the incident report gave the background of what happened on En. Fakhri’s machine.

Installation guide summary

  • The disk image was large, so downloading it through a browser was not ideal.
  • A browser download could fail if the connection dropped midway through.
  • Speed could be slower, and restarting a failed large download would be inconvenient.
  • Because of that, using a torrent client was the better approach.

Incident report summary

  • The affected user was En. Fakhri.
  • His files were encrypted by ransomware.
  • The important document mentioned was BigClient_Proposal_2025.docx.
  • Before the incident, he downloaded a compressed .rar archive, extracted it, got a resume template, and then deleted the archive.
  • The IT security team created a forensic image of the workstation for investigation.

Step 3: Download the disk image

Since the evidence image was around 10 GB, I installed qBittorrent on my Kali virtual machine and used the supplied torrent file to begin downloading the image.

sudo apt install qbittorrent qbittorrent DISKIMG_FAKRI251211.zip.torrent &

After running the commands, I waited for the download to complete. This step was mostly preparation, but it was necessary before any deeper analysis could begin.

Step 4: Calculate the SHA-256 value

Once the evidence file had been downloaded successfully, I calculated its SHA-256 hash. This directly answered the challenge requirement.

sha256sum FAKHRIWORKSTATION_20251211.E01

The output produced the hash value of the image, which was also the required flag for this challenge.

Final answer

Flag
nexsec25{c8f31718462337b4cc8218c2ca301ca9ca6122cca71c708757f38788533ca076}

Takeaway

Even though this first task was simple, it set up the rest of the OhMyFiles challenge series nicely. It introduced the victim context, showed that the case involved a ransomware incident, and reminded me that evidence handling often starts with very basic but important steps such as acquiring the image properly and verifying it with a hash.
